当你要求你的 AI Agent *"帮我看一下贵州茅台的资产负债表"* 或者是 *"批量下载中信证券最近5年的年度报告"* 时——在底层到底发生了什么？

如果没有合适的基础设施，大模型驱动的 Agent 要么凭空捏造（产生幻觉），要么试图一次性吞下数十万 Token 的 API 文档然后由于丢失关键上下文而跑偏，要么干脆直接报错放弃。

**A-Stock-Skills 正是为了解决这个问题而生。** 这是一个精心设计的 Agent Skill 集合，为你的 AI 提供了一张探索中国金融数据宇宙的结构化地图。它一改简单粗暴的“把文档全都扔给大模型”的做法，建立了一套基于**渐进式披露（Progressive Disclosure）**原则的智能导航系统。

---

## 解决现有方案的痛点

市面上的 AKShare 和 Tushare 拥有极为全面的官方接口能力，但如果你直接把它们包装成 Agent Skill，往往会陷入一个令人头疼的两难境地：

- **太浅**：只给 Agent 提供一小撮常用的接口示例或者简短的函数名描述，当用户的研究任务变得稍微深入和复杂时，Agent 无从下手。
- **太深**：直接打包所有库的内部说明给 Agent。在写出哪怕第一行代码前，你的上下文窗口余量与处理精度已经被大量的干扰信息消耗殆尽。

**A-Stock-Skills 开辟了第三条路：渐进式披露。**

触发技能时，Agent 首先获取的是一个高信息密度的入口（领域速查表、快速业务映射表）。只有在它明确了用户要做什么（例如：宏观分析、期货查询还是个股财务数据）之后，它才会根据指引按图索骥，去阅读特定的、小体积的参考文件以获得精准的参数。
结果显而易见：**极其精准的接口选择和代码执行，同时保持了极低的 Token 成本。**

### `akshare` — 全面映射的金融接口地图

AKShare 的数据极其庞大，涵盖 A/港/美股、宏观经济、期货期权、债券、ETF 和外汇等方方面面。本 Skill 将这种庞大的工程广度简化提炼成了让大模型极易遵循的 **5 步工作流**：

1. **识别域**：查阅域速查表，确定需求属于哪个业务域。
2. **扫描目录**：在 1,086 个接口的一行速查目录 (`references/catalog.md`) 中定位函数名。
3. **查阅详情**：深入对应的 `references/domains/<domain>.md` 获取极为准确的参数填法与返回字段。
4. **查阅模式**：参考 `references/patterns.md` 避开常见的请求频率限制或者日期参数陷阱。
5. **执行代码**

你的研究 Agent 终于不用再“猜”某个神秘的字段是用 `_sh` 还是用 `_SZ` 结尾了。

**支持领域的缩影：**

| 域 (Domain) | 接口数 | 核心覆盖数据 |
|---|---|---|
| `stock` | 403 | A/港/美股的行情与三大财务报表、北向资金流向、板块动态、龙虎榜数据 |
| `macro` | 226 | 中国 CPI/PPI/PMI/GDP、借贷利率，美联储利率、非农就业及全球宏观基本面 |
| `index` | 80 | 申万/中证/国证/全球指数大盘，QVIX 恐慌指数估算 |
| `fund` | 73 | ETF/LOF/开放式公募基金净值走势、持仓明细跟踪 |
| `futures` | 67 | 国内四大期交所及外盘的主力/连续合约，每日仓单与库存变化 |
| `bond` | 42 | 国债基准收益率直连、可转债（含强赎/下修条款数据追踪）|
| `option` | 47 | 场内外股票期权与商品期权报价表与希腊字母风险 |
| `spot/fx/energy` | 50+ | 黄金等大宗商品现货、央行外汇牌价基准与碳中和排碳资产行情 |

**支持领域的缩影（共 220+ 核心接口）：**

|---|---|---|

| `finance` | 10+ | 深度清洗的上市公司三大核心报表、业绩预告、主营业务构成与财务指标矩阵 |
| `index` | 18+ | 涵盖沪深核心大盘至申万/中信行业指数的成分股权权及每日、分钟级切片行情 |
| `fund/etf` | 16+ | 场外公墓清单列表与净值、历史持仓穿透；ETF 行情与高频的份额/规模流入流出变动 |

### `cninfo-report` — 自动化财报批量提取引擎

这是目前市面上极其少有的一种专门为研报解析场景打造的基础工具：它赐予你的 Agent 直接从深交所旗下官方信息平台——**巨潮资讯网**，自动化搜索并批量下载原始 PDF 财务披露声明的能力。

你只需要用人类语言下达命令，Agent 便会组合出并静默运行相应的终端脚本：

```bash
# 自动抓取“股王”最近 5 年的所有年度报告
python cninfo-report.py 600519

# 跨市场搜索并下载：腾讯（港股）近 3 年报告
python cninfo-report.py 00700 --market 港股 --years 3

# 无死角打包：平安银行近 10 年的所有季报、半年报和年报
python cninfo-report.py 000001 --years 10 --type all
```

所有的研究资料都会被 Agent 井井有条地自动归档入库：
```
财务报告下载/
└── 600519_贵州茅台_年度报告/
    ├── 600519_贵州茅台_20240330_贵州茅台2023年年度报告.pdf
    └── ...
```

（原生支持**断点续传**防网络中断，再次运行指令会自动跳过已经落盘的文档）

## 设计哲学与愿景

**请牢记：A-Stock-Skills 是基础设施层（Infrastructure），而不是顶层应用产品。**

它的任务不是去给股票估值，也不会直接帮你写几千字的宏观对冲研究报告。它的唯一神圣使命是使得你的 Agent 可以100% 可靠、精准和高速地 **找到对应的接口并把冷冰冰的数据捞上来**，然后把舞台让出来。接下来多智能体协同、思维链推理（CoT）或是精巧的投资分析流转，全部交由你的上层系统接管。

在此之下，该项目的每一个字节都在遵守这三个戒律：

1. **渐进式披露至上** — 坚壁清野的入口层，辅之以深不可测的技术细节库随取随用。
2. **面向 Agent 优化** — 数据字典、枚举与工作流排版完全是由于大语言模型擅长的处理模式而妥协（如：放弃折叠，极简矩阵），并不是为了人类双眼的阅读习惯而排版。
3. **零容忍系统性幻觉** — 对于关键的财务因子拼接和神秘字符要求，AI 在这里的准则是“依宪执行，不可凭空乱造”，它必须从知识库读取而不是瞎猜映射。
